Yeah, it isn’t THAT weird OOP stands for “Object Oriented Programming” and it is a programming paradigm in which you write your code into encapsulated blocks of code called classes which are basically blueprints that determine how objects behave. Objects are then bits of code that have a clear purpose and structure and they can access specific data and methods. Let’s think of an everyday object, a door. A door has many attributes, for example it’s color, it’s material, maybe it’s weight, it also has a property, is it open or closed, and it has a method, open or close.
